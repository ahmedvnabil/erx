# خطة إكمال ERX وتوسيع المصادر

## الهدف

الوصول من 24 مصدرًا نشطًا و929 وثيقة قابلة للبحث إلى **80 مصدرًا نشطًا
على الأقل** و20,000 وثيقة موثقة، دون خفض جودة الاسترجاع أو كسر سياسات
`robots_policy=respect` و`crawl_delay` أو حقول citation.

لا يُحسب المصدر ضمن الهدف لمجرد وجوده في الكتالوج؛ يجب أن يجتاز بوابة القبول
ويظل سليمًا سبعة أيام متصلة.

## تعريف الاكتمال

- Precision@5 ≥ 0.80، Recall@20 ≥ 0.85، nDCG@10 ≥ 0.80، MRR ≥ 0.85.
- اكتمال التحكيم اليدوي لـ30–50 استعلامًا مع خمسة أحكام موجبة ممكنة لكل استعلام.
- لا تظهر نتيجة تحت عتبة الصلة، ورسالة عربية واضحة عند غياب المطابقات القوية.
- أعلى عشر نتائج سياسية تضم ثلاثة أنواع مصادر عند توفرها.
- 90% على الأقل من الوثائق القابلة للبحث موسومة بموضوع واحد.
- كل نتيجة تحتفظ بـ`title/source/url/published_at/archived_at`.
- 80 مصدرًا نشطًا، نجاح جمع ≥95%، ولا مصدر يخالف robots أو الترخيص.
- 80%+ تغطية اختبارات، build وaudit وrelease check وE2E كلها خضراء.

## بوابة قبول أي مصدر

1. **الهوية:** ناشر معروف، URL أصلي، `source_type` و`ownership_type` موثقان.
2. **القانون والأخلاق:** robots قابل للجمع، الترخيص/الاستخدام مسجل، وتأخير الزحف
   لا يقل عن القيمة المنشورة أو 2 ثانية عند غيابها.
3. **الاستخراج:** عنوان وتاريخ ونص ورابط canonical صالح؛ لا صفحات قوائم أو إعلانات.
4. **الاستشهاد:** `archived_at` إلزامي، مع حفظ نسخة/بصمة حيث يسمح الترخيص.
5. **الجودة:** 20 وثيقة عينة بدقة استخراج ≥95% وduplicate rate ≤5%.
6. **التشغيل:** نجاح يومي سبعة أيام، ثم تفعيل backfill المتدرج.
7. **الفشل الآمن:** المصدر غير الصالح يبقى `catalog_only` مع سبب الاستبعاد، ولا يُحذف تاريخه.

## ترتيب التنفيذ

### المرحلة 0 — إصلاح القياس (يومان)

**T0.1 — استكمال gold set**

- pool أعلى 20 نتيجة من lexical وsemantic والنسخة الحالية لكل استعلام.
- تحكيم يدوي مزدوج، وحسم الخلاف بواسطة مراجع ثالث.
- منع إصدار تقرير P@5 إذا كان سقف الأحكام أقل من هدف القبول.

**القبول:** 30–50 استعلامًا عربيًا، ≥5 وثائق موجبة حين يسمح الكوربوس، وتقرير baseline قابل للتكرار.

### المرحلة 1 — إنهاء محرك الاسترجاع A (3–4 أيام)

**T1.1 — BM25 عربي وحقول موزونة**

- وزن العنوان والكيانات والموضوعات أعلى من النص.
- phrase/proximity bonus، وتطبيع عربي وأرقام وstopwords قبل الفهرسة والاستعلام.

**T1.2 — RRF وreranking وعتبة معايرة**

- دمج BM25 والدلالي بـRRF، مع منع semantic-only غير المؤيد لفظيًا.
- معايرة العتبة على validation split منفصل وإضافة تفسير ترتيب مستقر.

**القبول:** كل مقاييس الاسترجاع تحقق تعريف الاكتمال، واستعلام التضخم بلا ضجيج عسكري.

### المرحلة 2 — التنوع B والموضوعات C (4–5 أيام)

**T2.1 — balanced mode**

- MMR/حصص مدركة لـ`source_type` و`ownership_type`، دون رفع وثيقة غير ذات صلة.
- اختبارات سياسية تضمن ثلاثة أنواع مصادر عندما توجد تغطية.

**T2.2 — تصنيف 16 موضوعًا**

- classifier عربي متعدد الوسوم مع confidence ونسخة model/rules.
- backfill قابل للاستئناف، وتقارير تغطية وأخطاء.

**T2.3 — الكيانات والروابط**

- aliases عربية، إزالة الالتباس، وربط الأشخاص والمؤسسات والأماكن والقوانين.
- إعادة بناء timeline وclaims بعد backfill.

**القبول:** topic coverage ≥90%، واختبارات التنوع والكيانات خضراء.

### المرحلة 3 — إعادة تأهيل 28 مصدرًا متقاعدًا (أسبوع)

تُراجع قبل إضافة مصادر جديدة لأنها معرفة بالفعل في الكود:

- **قانون/حكومة:** المطابع الأميرية، وزارة العدل، وزارة المالية، البنك المركزي،
  النيابة العامة، الرقابة المالية، الجمارك، البيئة، الدواء، تنظيم الاتصالات،
  تنظيم الكهرباء، الرقابة على الصادرات، وزارة التخطيط SDDS.
- **حقوق:** EIPR، AFTE، Amnesty Egypt، HRW Egypt، ICNL Egypt.
- **أخبار:** الأهرام، المصري اليوم، مصراوي، القاهرة 24، Daily News Egypt،
  Egypt Independent، Egyptian Streets.
- **أكاديمي:** معهد التخطيط القومي، BUE Scholar، MSA Repository.

لكل مصدر: audit → connector test → 20-document sample → seven-day canary → backfill.

**الهدف:** إعادة 18 مصدرًا على الأقل؛ البقية تظل catalog-only مع سبب موثق.

### المرحلة 5 — backfill آمن ومتدرج (5–7 أيام تشغيل)

- backfill حسب المصدر والسنة، بصف انتظار ومؤشر استئناف.
- حد أقصى افتراضي: طلب واحد كل ثانيتين، وأبطأ إذا طلب robots ذلك.
- exponential backoff وcircuit breaker بعد الأخطاء المتكررة.
- dedup بالـcanonical URL ثم content hash ثم near-duplicate.
- تقرير يومي: fetched/accepted/excluded/duplicate/error وسبب كل استبعاد.
- أولوية تاريخية: البرلمان، الدستورية، النقض، سيناء، RSF، الحقوق، الاقتصاد.

### المرحلة 6 — الرصد والجودة E (3 أيام)

- score distribution، no-strong-match rate، zero-result rate، latency p50/p95.
- topic/source/ownership distribution لكل استعلام وعبر الزمن.
- request ID في HTTP وMCP structured logs دون query text كامل افتراضيًا.
- alerts عند فشل مصدرين متتاليين أو انخفاض searchable yield أو تغير parser.
- صفحة coverage تعرض المصادر catalog-only والمتقاعدة وأسبابها.

### المرحلة 7 — بوابة الإصدار والنشر (يومان)

1. تشغيل full test وcoverage وaudit وrelease check وnpm pack.
2. اختبار MCP compatibility لكل الأدوات الحالية.
3. browser E2E للصفحة و`/sources` بالعربية وRTL على 320/768/1440.
4. backup موثّق وتجربة restore.
5. canary بنسبة محدودة، ثم production، ثم npm بنفس رقم الإصدار.
6. تحقق حي من `/healthz` و`/mcp` وثلاثة استعلامات gold.

## نقاط التوقف

- بعد المرحلة 1: لا توسع corpus قبل ثبات baseline ومنع الضجيج.
- بعد كل عشرة مصادر: rerun eval لمنع تدهور الدقة.
- لا backfill لمصدر يفشل robots أو extraction sample.
- لا نشر إذا كان أي معيار من تعريف الاكتمال أحمر.

## الناتج المتوقع

- **الحد الأدنى:** 60 مصدرًا نشطًا و10,000 وثيقة خلال أول دورتين.
- **الهدف:** 80 مصدرًا نشطًا و20,000 وثيقة موثقة.
- **الامتداد:** 100+ مصدر في الكتالوج، بشرط أن تظل المصادر غير المؤهلة
  catalog-only وألا تُستخدم لملء الرقم أو التأثير في البحث.

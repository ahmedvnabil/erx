# ERX product roadmap

## معيار الاعتماد

كل نتيجة عامة يجب أن تحمل المصدر الأصلي، وقت الجلب، نوع المصدر، ورابطًا قابلًا لإعادة التحقق. لا تُمنح الادعاءات درجة حقيقة آلية، ولا تُزال الوثائق عند تقاعد مصدر؛ يُسجل التغيير ويظل الأصل قابلًا للتتبع.

## الأولويات التنفيذية

1. تجميع القصص عبر مصادر مستقلة مع تفسير سبب التجميع.
2. كتالوج بيانات موحد للوثائق والبيانات الحية مع الترخيص والتواتر والحالة.
3. مراقبة الصحة والتغطية، cache، retries، وصفحة حالة عامة.
4. تنبيهات واستعلامات محفوظة وموجز يومي للباحثين.
5. SDKs لـ TypeScript وPython، أمثلة تحليلية، وWebhooks.

## سياسة المصادر

المسار العام يقبل المصادر العامة القابلة للتحقق فقط. المصادر التي تتطلب Token أو حسابًا خاصًا تبقى خارج الكتالوج العام أو تظهر كـ `catalog_only` مع توضيح السبب. البيانات الحساسة، خصوصًا البيانات الفردية، لا تُعرض؛ تُستخدم التجميعات المنشورة فقط.

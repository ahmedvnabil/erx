# نموذج التهديد

## الأصول

- سلامة الأرشيف وسلسلة الاستشهاد والنسخ التاريخية.
- مفاتيح مزودي embeddings ومفتاح الإدارة.
- توافر البحث وMCP أمام المستخدمين.
- عدم تسريب بيانات تشغيل أو مسارات داخلية.

## حدود الثقة وإساءة الاستخدام

| الحد | التهديد | الضابط |
|---|---|---|
| رابط من RSS/Sitemap | SSRF أو redirect إلى شبكة خاصة | نفس المضيف، DNS عام، منع redirects غير المسموحة |
| HTML/PDF خارجي | ملف ضخم أو parser bomb أو prompt injection | حد bytes/pages/time، subprocess بلا shell، النص بيانات فقط |
| استعلام البحث | SQL/FTS injection أو استهلاك مفرط | tokenization، parameterization، حدود طول ونتائج ومعدل |
| REST/MCP العام | DoS أو scraping عدائي | rate limit، timeouts، response caps، metrics |
| الإدارة | انتحال أو تشغيل جمع مكلف | مفتاح إدارة من البيئة ومقارنة ثابتة الزمن وسجل تدقيق |
| مزود embedding | تسريب محتوى أو مخرجات تالفة | opt-in، إرسال مقتطف محدود، تحقق الأبعاد والقيم |

## قرارات

- لا توجد حسابات مستخدمين أو كلمات مرور في هذه المرحلة؛ القراءة عامة والإدارة
  بمفتاح بيئي. إضافة OAuth لاحقًا تحتاج مزودًا ودومينًا محددين.
- لا توجد file uploads عامة. ملفات PDF تُجلب فقط من مصادر معرفة مسبقًا.
- لا تنفذ المنصة تعليمات موجودة داخل محتوى مؤرشف أو مخرجات نموذج.
- لا يصبح تكرار الادعاء درجة حقيقة؛ الأدلة تعرض حسب المصدر والاستقلال.
- HSTS يُفعّل فقط خلف HTTPS في الإنتاج، وبقية رؤوس الأمان دائمًا.

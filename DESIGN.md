# تصميم الاسترجاع في ERX

## الميثاق

ERX يعيد وثائق وأدلة منسوبة إلى مصادرها، ولا يحوّل تكرار الادعاء أو درجة
الاسترجاع إلى حكم حقيقة. يبقى الاستشهاد كاملًا: العنوان، المصدر، الرابط، تاريخ
النشر، وتاريخ الأرشفة.

## خط الترتيب

1. يُطبّع الاستعلام عربيًا: الألف والهمزات، الياء والألف المقصورة، التاء
   المربوطة، التشكيل، التطويل، والأرقام العربية-الهندية.
2. تُحذف stopwords العربية، ثم تُولّد صيغ أل التعريف واللواصق والجمع.
3. تُستخرج قائمة BM25 صارمة للاستعلام وقوائم مرشحين لكل مصطلح مهم.
4. تُدمج القوائم بـRRF (`k=60`) مع وزن لفظي `1.5` ووزن دلالي `0.35`.
5. لا تدخل نتيجة دلالية وحدها؛ يلزم تطابق لفظي مع نصف مصطلحات الاستعلام على
   الأقل. هذه قاعدة abstention مقصودة للكوربوس الصغير.
6. يعاد الترتيب بزيادة لتطابق العنوان والموضوع، ثم تطبق عتبة إلزامية
   `MIN_RETRIEVAL_SCORE=0.018`.
7. `match_reasons` يوضح إن كانت الإشارة لفظية أو دلالية، ولا يمثل درجة حقيقة.

## التقييم

`eval/gold-set.json` يحتوي 30 استعلامًا عربيًا وأحكام صلة متدرجة مرتبطة بمعرفات
وثائق لقطة الإنتاج `production-v0.16.2-2026-07-18`. يشغّل:

```bash
npm run eval:retrieval -- data/research.db
```

المقاييس هي Precision@5 وRecall@20 وnDCG@10 وMRR. يجب توسيع pooled judgments
كلما أضيفت مصادر أو وثائق جديدة، ولا يجوز اعتبار وثيقة غير محكّمة غير ذات صلة
تلقائيًا في التقارير المنشورة.
